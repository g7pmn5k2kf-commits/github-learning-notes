# 01 - 基礎概念

## Git 和 GitHub 的差別

| | 說明 | 比喻 |
|---|---|---|
| Git | 裝在電腦裡的工具，負責記錄每一次修改 | Word（編輯） |
| GitHub | 存放 Git 記錄的雲端網站 | Google Drive（存放） |

兩個要搭配使用：Git 追蹤修改，GitHub 備份到雲端。

---

## Repository（倉庫）是什麼

一個專案就是一個 Repository（簡稱 Repo）。
可以把它想成一個「有記憶的資料夾」——不只存放檔案，還記錄每一次修改的歷史。

---

## Commit 是什麼

Commit 是一個「存檔點」，記錄這次修改了什麼。

**Commit 包含三件事：**
- 這次改了什麼內容（diff）
- 什麼時間改的
- Commit message：你自己寫的說明

**Commit message 怎麼寫才好？**
三個月後看到這筆記錄，要能馬上知道當時做了什麼。

- 壞例子：`修改`
- 好例子：`新增這個 Repo 的用途說明`

**Commit 不只是記錄，也是可以回去的點。**
如果改壞了，可以跳回任何一個之前的 Commit，把檔案還原到那個狀態。

---

## 版本歷史（Commits 頁面）

在 Repo 頁面點 Commits 數字，可以看到所有存檔點的清單。
每筆記錄旁邊有一串英數字（例如 `3e0b532`），叫做 **Commit ID**，是每個存檔點的唯一編號。

---

## Diff（差異比較）

點進任一個 Commit，可以看到這次修改的詳細內容：

- 綠色 `+` 號：這次新增的內容
- 紅色 `-` 號：這次刪除的內容

GitHub 不是存整份檔案，而是只記錄「每次改了什麼」。
