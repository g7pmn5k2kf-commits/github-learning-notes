# 05 - 下載開源專案：Clone vs Fork

## 三種取得別人專案的方式

| 方式 | 說明 | 適合情況 |
|------|------|----------|
| Fork | 複製一份到自己的 GitHub 帳號 | 想保留或修改原專案 |
| Clone | 把 repo 下載到自己電腦 | 想在本機使用或編輯 |
| Download ZIP | 下載壓縮檔 | 只是想看內容，不需要版本控制 |

---

## Fork 是什麼

Fork 是把別人的 repo 複製一份到你自己的 GitHub 帳號。

- Fork 後的 repo 完全屬於你，可以自由編輯
- 原作者看不到你的修改
- Fork 後的頁面會顯示 forked from 原作者/repo名稱

---

## Sync fork 是什麼

原作者更新了他的 repo 之後，你可以按 Sync fork 把更新同步過來。

注意事項：
- 你改了 A 檔案，原作者改了 B 檔案 → 兩邊修改都保留，不衝突
- 你和原作者改了同一行 → GitHub 會提示衝突，需要手動決定保留哪個版本

---

## Clone 是什麼

Clone 是把 repo 下載一份到你的電腦裡。

- 用 GitHub Desktop 操作最簡單
- 下載後可以在本機編輯，完成後 Push 回 GitHub
- 以後原版更新，可以同步到最新版本

---

## Download ZIP 和 Clone 的差別

- Download ZIP：死的一份，沒有版本控制，原版更新你不知道
- Clone：活的一份，有版本控制，可以隨時同步更新

---

## 如何判斷用哪種方式

- 只是想看或用別人的專案 → Clone 或 Download ZIP
- 想保留一份在自己帳號、或想修改後回傳給原作者 → Fork

---

## 開源專案頁面的重要數字

| 數字 | 說明 |
|------|------|
| Star | 收藏數，越高代表越受歡迎 |
| Fork | 有多少人複製了這個專案 |
| Watch | 有多少人訂閱這個專案的更新通知 |
