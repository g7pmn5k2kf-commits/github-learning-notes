# 02 - 第一個 Repo

## 建立 Repository 的步驟

1. 點右上角 `+` 號
2. 選 `New repository`
3. 填入 Repository name
4. 勾選 `Add a README file`
5. 點 `Create repository`

---

## README.md 是什麼

README 是這個 Repo 的「說明書」，讓人一眼看懂這個專案是做什麼的。
每個 Repo 的首頁都會自動顯示 README 的內容。

---

## 在網頁上編輯檔案

GitHub 不需要任何軟體，直接在瀏覽器上就能編輯檔案。

1. 點檔案右邊的鉛筆圖示進入編輯模式
2. 修改內容
3. 點右上角 `Commit changes...` 儲存

---

## Commit 的操作流程

點 `Commit changes...` 後會跳出一個視窗：

| 欄位 | 說明 |
|------|------|
| Commit message | 這次修改的說明（必填） |
| Extended description | 更詳細的說明（選填） |
| Commit directly to main | 直接存到主線 |

填好 Commit message 後按 `Commit changes` 完成儲存。

---

## 版本歷史

在 Repo 主頁點 Commits 數字，可以看到所有存檔記錄。

每一筆記錄包含：
- Commit message（當時寫的說明）
- 時間
- Commit ID（例如 `3e0b532`）

---

## Diff（差異比較）

點進任一個 Commit，可以看到這次「改了什麼」：

- 綠色 `+`：新增的內容
- 紅色 `-`：刪除的內容

---

## Blame 視圖

在檔案頁面點 `Blame` 標籤，可以看到每一行是「誰」在「什麼時候」透過哪個 Commit 寫的。
