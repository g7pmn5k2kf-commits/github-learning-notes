# 06 - 進階操作與實戰練習

## Issues 是什麼

Issues 是 GitHub 內建的問題追蹤系統，可以用來記錄：
- 待辦事項
- 已知的 bug
- 新功能需求
- 想問作者的問題

就算自己一個人用，也可以當作專案的備忘錄。

---

## Issues 的狀態

| 狀態 | 說明 |
|------|------|
| Open | 還未解決或完成 |
| Closed | 已解決或完成 |

處理完一個 Issue 後，把它關閉（Close），就會移到 Closed 清單。

---

## Star 和 Watch 的差別

| 功能 | 說明 | 適合情況 |
|------|------|----------|
| Star | 收藏專案，像書籤 | 覺得有用但不常看 |
| Watch | 訂閱通知，有更新就收到通知 | 正在積極使用的專案 |

---

## 看懂陌生開源專案的步驟

1. 看 README — 了解這個專案是做什麼的
2. 看資料夾結構 — 大概了解專案分成哪些部分
3. 看 Commits 數量和更新時間 — 判斷專案是否還在維護
4. 看 Issues — 了解已知問題和其他人的使用情況
