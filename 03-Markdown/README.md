# 03 - Markdown 語法

## Markdown 是什麼

Markdown 是一種輕量的標記語言，用簡單的符號來控制文字排版。
GitHub 上所有的 README.md 都用這個格式撰寫。

---

## 常用語法對照表

| 語法 | 說明 |
|------|------|
| # 文字 | 大標題 |
| ## 文字 | 第二層標題 |
| ### 文字 | 第三層標題 |
| - 文字 | 圓點清單 |
| **文字** | 粗體 |
| 普通文字 | 直接顯示為段落 |

---

## 標題層級

標題用井號 # 開頭，井號越多標題越小，# 後面一定要加一個空格。

範例：
    # 第一層（最大）
    ## 第二層
    ### 第三層

---

## 清單

用 - 加空格開頭，就會變成圓點清單。

範例：
    - 項目一
    - 項目二
    - 項目三

---

## 粗體

文字前後各加兩個星號，就會變成粗體。

範例：**這是粗體**

---

## 表格

用 | 分隔欄位，第二行用 --- 分隔標題和內容。

範例：
    | 欄位一 | 欄位二 |
    |--------|--------|
    | 內容一 | 內容二 |

---

## 勾選清單

用 - [x] 表示已完成，- [ ] 表示未完成。

範例：
    - [x] 已完成的項目
    - [ ] 未完成的項目

---

## 重要觀念

- 符號要在文字前面才有效果
- # 後面一定要加一個空格
- - 後面一定要加一個空格
- 用 Preview 隨時確認排版是否正確
